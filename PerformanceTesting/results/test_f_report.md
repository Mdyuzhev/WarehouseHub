### Ключевые наблюдения

1. **Пик производительности: 150 users / 63 RPS**
   - При 150 users система показывает максимальный RPS
   - Latency остаётся приемлемой (p95 < 1000ms)
   - 0% ошибок

2. **Точка насыщения: 200-250 users**
   - RPS начинает падать (60 → 47)
   - Latency значительно растёт (1700ms → 4200ms)
   - Появляются первые единичные ошибки

3. **Деградация: 350+ users**
   - Error rate растёт экспоненциально
   - RPS падает (37-54)
   - p95 превышает 7-12 секунд

4. **Критическая точка: 400 users**
   - Error rate > 2% (достигнут порог)
