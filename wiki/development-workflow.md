# Автоматизированный Workflow разработки

## Обзор

Данный документ описывает автоматизированный процесс разработки, реализованный в проекте warehouse-api. Процесс интегрирует YouTrack (issue tracking), GitLab (CI/CD) и Claude Code агента (автоматизация) в единый workflow.

## Процесс разработки

Workflow состоит из пяти последовательных этапов, каждый из которых автоматизирован через Claude Code агента.

### Этап 1: Создание задачи в YouTrack

Агент создаёт задачу через REST API YouTrack. Задача содержит структурированное описание с целью, планом работ и критериями приёмки.

```bash
curl -X POST "http://192.168.1.74:8088/api/issues?fields=id,idReadable,summary" \
  -H "Authorization: Bearer <YOUTRACK_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "project": {"id": "0-1"},
    "summary": "Название задачи",
    "description": "## Цель\n\nОписание цели...\n\n## План\n\n1. Шаг 1\n2. Шаг 2"
  }'
```

### Этап 2: Планирование в задаче

В описании задачи фиксируется детальный план работ в формате Markdown. Это обеспечивает прозрачность и возможность отслеживания прогресса.

### Этап 3: Выполнение работы

Агент выполняет технические задачи: создаёт файлы, модифицирует код, запускает команды на сервере через SSH. Все действия соответствуют плану из задачи.

### Этап 4: Документация в Wiki

После выполнения работы агент создаёт документацию в директории `wiki/` проекта. Документация содержит описание решения, команды, примеры использования и troubleshooting.

### Этап 5: Коммит и отчёт

Агент коммитит изменения с указанием номера задачи в сообщении (например, `#WH-17`), что автоматически связывает коммит с задачей в YouTrack. Затем добавляет комментарий в задачу с отчётом о выполнении.

```bash
# Коммит с привязкой к задаче
git commit -m "#WH-17 Описание изменений"
git push origin main

# Комментарий в YouTrack
curl -X POST "http://192.168.1.74:8088/api/issues/WH-17/comments" \
  -H "Authorization: Bearer <YOUTRACK_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"text": "Задача выполнена. Коммит: abc123"}'
```

## Настройка интеграции

### Получение YouTrack API Token

Для работы с YouTrack API необходим Permanent Token.

1. Открыть YouTrack: http://192.168.1.74:8088
2. Перейти в Profile → Account Security
3. В разделе Tokens нажать New token
4. Указать имя токена (например, `claude-agent`)
5. Scope: YouTrack (полный доступ)
6. Нажать Create и сохранить токен

Токен имеет формат: `perm:BASE64_USER.BASE64_ID.SECRET`

### API Endpoints

Основные endpoints YouTrack API, используемые в workflow:

| Операция | Метод | Endpoint |
|----------|-------|----------|
| Список проектов | GET | /api/admin/projects |
| Создание задачи | POST | /api/issues |
| Получение задачи | GET | /api/issues/{id} |
| Добавление комментария | POST | /api/issues/{id}/comments |
| Изменение статуса | POST | /api/issues/{id} |

### Интеграция GitLab + YouTrack

YouTrack автоматически связывает коммиты с задачами по паттерну `#WH-XX` в сообщении коммита. Для этого настроена VCS Integration в YouTrack.

## Преимущества автоматизированного Workflow

Внедрение автоматизированного процесса даёт несколько ключевых преимуществ.

**Трассируемость.** Каждое изменение в коде связано с задачей в YouTrack. История разработки полностью документирована.

**Консистентность.** Все задачи создаются по единому шаблону с целью, планом и критериями приёмки. Это упрощает оценку и приёмку работ.

**Скорость.** Автоматизация рутинных операций (создание задач, коммитов, комментариев) экономит время разработчика.

**Документация.** Каждая задача сопровождается документацией в wiki, что формирует базу знаний проекта.

**Интеграция.** Связка YouTrack + GitLab + Claude Code работает как единая система, где информация автоматически синхронизируется между компонентами.

## Команды для работы с YouTrack API

### Получить список проектов

```bash
curl -X GET "http://192.168.1.74:8088/api/admin/projects?fields=id,name,shortName" \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Accept: application/json"
```

### Создать задачу

```bash
curl -X POST "http://192.168.1.74:8088/api/issues?fields=id,idReadable,summary" \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "project": {"id": "0-1"},
    "summary": "Название задачи",
    "description": "Описание в Markdown"
  }'
```

### Добавить комментарий

```bash
curl -X POST "http://192.168.1.74:8088/api/issues/WH-XX/comments" \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"text": "Текст комментария"}'
```

### Получить информацию о задаче

```bash
curl -X GET "http://192.168.1.74:8088/api/issues/WH-XX?fields=id,summary,description,comments(text)" \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Accept: application/json"
```
