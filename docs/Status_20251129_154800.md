## Решённые проблемы

### 1. Redis port parsing error
**Проблема:** K8s создаёт переменную `REDIS_PORT=tcp://IP:port` которая конфликтует с Spring `spring.data.redis.port`

**Решение:** Захардкожены значения в `application-k8s.properties`:
```properties
spring.data.redis.host=redis
spring.data.redis.port=6379
```

### 2. Redis serialization error
**Проблема:** `SerializationException: Cannot serialize` для Product

**Решение:** Добавлен `implements Serializable` в модель Product (коммит `70237b4`)

### 3. Telegram Bot 409 Conflict
**Проблема:** Конфликт polling при RollingUpdate

**Решение:**
- `strategy: Recreate` в deployment
- Graceful shutdown с signal handlers
- `terminationGracePeriodSeconds: 15`
